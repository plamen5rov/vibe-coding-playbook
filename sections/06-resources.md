## Staying current

The vibe coding tooling changes fast. These are the most reliable signals:

| Source | What to watch | URL |
| --- | --- | --- |
| OpenCode releases | New features, breaking changes | <https://github.com/anomalyco/opencode/releases> |
| Anthropic news | Claude Code announcements | <https://www.anthropic.com/news> |
| Claude Code changelog | Detailed release notes | <https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md> |
| r/ClaudeAI | Community tips, issue reports | <https://www.reddit.com/r/ClaudeAI/> |
| r/vibecoding | General community discussion | <https://www.reddit.com/r/vibecoding/> |
| Anthropic on X | Real-time announcements | <https://x.com/AnthropicAI> |
| OpenCode Discord | OpenCode-specific discussion | Linked from opencode.ai |

**GitHub Watch tip:** Star and watch the `anomalyco/opencode` and `anthropics/claude-code` repos. Set notifications to "Releases only" to get notified when new features drop without inbox noise.

## Context engineering — going deeper

Vibe coding evolved from "describe what you want" to "context engineering" — the deliberate structuring of information the agent receives. If you want to go deeper:
